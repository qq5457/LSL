灵兽录/
├── index.html                  # 主页面（只保留 HTML 结构，引入样式和入口模块）
├── styles/
│   └── main.css                # 全部 CSS（从 <style> 中搬出）
├── scripts/
│   ├── config.js               # 游戏常量与静态数据（MAPS、技能池、公式等）
│   ├── utils.js                # 纯工具函数（randInt、choice、uid 等）
│   ├── storage.js              # 存档读写抽象层（封装 window.storage 或 localStorage）
│   ├── state.js                # 游戏状态创建、初始化、导入导出、版本兼容
│   ├── combat.js               # 战斗模拟、遭遇生成、捕获判定、掉落处理
│   ├── pet.js                  # 宠物对象创建、属性计算、升级/经验、突破（未来）、技能相关
│   ├── ui.js                   # 所有 DOM 渲染、事件绑定、UI 更新逻辑
│   └── main.js                 # 入口：组装模块、初始化游戏、启动循环
└── data/                       # (预留) 未来可放配置 JSON，比如地图、技能表等

模块职责详解
文件	对应原代码区域	核心内容
index.html	<body> 内所有结构	只保留界面骨架，<link> 引用 main.css，<script type="module" src="scripts/main.js"> 作为入口
styles/main.css	<style> 内全部 CSS	全部样式，方便以后调整视觉
scripts/config.js	顶部 MAPS、SKILL_POOL、GROWTH_BASE、BASE_STATS、TIER_LABEL 等	所有不变的游戏数据、平衡常数、文本定义
scripts/utils.js	randInt、choice、clamp、uid 等函数	无状态的纯工具函数，不依赖其他模块
scripts/storage.js	saveGame、loadGame、flashSaveStatus、SAVE_KEY	封装存档读写，当前使用 window.storage，未来想改 localStorage 只改这里
scripts/state.js	freshState()、applySaveDefaults()、importSaveFromText()、exportSave()、pad2()	状态生成、导入导出、版本兼容处理；持有全局 state 对象
scripts/combat.js	simulateBattle、resolveEncounter、generateEncounter、rollBaby、buildCreature、generateEquipment、handleOfflineProgress 等	所有战斗、遭遇、掉落、捕获、离线结算逻辑；引用 config、utils、state、pet
scripts/pet.js	computeStats、creatureToPet、grantPetExp、expToNext、playerExpToNext、soulValue、catchChance、rollTalent、rollSkills 等	宠物属性计算、升级、品阶相关的纯函数，未来 2.0 的突破、等级上限都加在这里
scripts/ui.js	所有 render* 函数、bindEvents、pushLog、openPetDetail、trainPet、refinePet 等交互函数	完全负责界面更新、事件绑定、用户操作响应；依赖 state、pet、combat、storage
scripts/main.js	init()、计时器、gameTick、startExploreLoop、stopExploreLoop、debounceSave、定时存档等	组装所有模块，启动游戏，协调定时循环
